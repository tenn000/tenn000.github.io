Hello 算法

算法分析

主要通过时间效率（算法运行速度快慢）和空间效率（算法占用内存大小）两个维度来评价算法的效率，而评估这两个效率的高低用到了复杂度分析。

复杂度因评估时间和空间两个维度的效率，而分为时间复杂度和空间复杂度；复杂度与输入数据量是有关的；复杂度是一种趋势性分析。

时间复杂度：

计算渐进上界就是$找一个函数f(n)，使得在趋于∞时，T(n)和f(n)处于相同增长级别（仅相差一个常数项c的倍数）$

例如，

$T(n)=3n+2，那么f(n)=n$

函数的渐近上界用大O记号来表示。

上述函数$T(n)的时间复杂度为O(T(n))=O(f(n))=O(n)$

这里的$O(f(n))$是这样计算的

  • 忽略数量与n无关的操作
  • 忽略系数
  • 循环嵌套时使用乘法

通常线性阶出现于单层循环；平方阶出现于嵌套循环；阶乘阶和指数阶通常出现在递归函数中；对数阶通常出现在使用二分查找、分治算法等的递归函数中；线性对数阶通常出现在主流排序算法中。

空间复杂度：

常数阶常见于数量与输入数据大小n无关的常量、变量、对象；线性阶常见于元素数量与n成正比的数组等数据结构中；指数阶常见于二叉树；对数阶常见于分治算法等。

以空间换时间最为常用。

数据结构简介

基本数据类型就是CPU可以直接运算的类型，能直接使用而不需要构造。整数 byte, short, int, long 、浮点数 float, double 、字符 char 、布尔 boolean 是计算机中的基本数据类型，占用空间的大小决定了它们的取值范围。

数据结构可根据逻辑结构和物理结构进行分类。

  • 逻辑结构

    • 线性：数组、链表、栈、队列、哈希表
    • 非线性：树、图、堆、哈希表

    为什么两种类型里都存在哈希表呢？

  • 物理结构

    • 连续空间存储（数组）：栈、队列、哈希表、树、堆、图、矩阵、张量（维度 ≥3 的数组）等；
    • 离散空间存储（链表）：栈、队列、哈希表、树、堆、图等；

数组与链表

数组

关键是数组中元素的插入与删除。

  • 数组插入元素

    在索引i处插入一个元素，需要将索引i及以后的所有元素后移一位，再给i处赋值。

  • 数组删除元素

    删除索引i处的元素，只需要把索引i之后的元素都向前移一位。删除元素后，原先末尾的元素变得“无意义”了，我们无需特意去修改它。

链表

链表结点的数据结构：

/* 链表结点结构体 */
struct ListNode {
    int val;         // 结点值
    ListNode *next;  // 指向下一结点的指针（引用）
    ListNode(int x) : val(x), next(nullptr) {}  // 构造函数
};

链表的初始化分为两步，首先要初始化各个结点对象，然后要构建引用指向关系。

  • 链表中插入结点：

    在结点n0和n1之间插入结点p，只需要使得n0->next指向p，p->next指向n1即可。

    /* 在链表的结点 n0 之后插入结点 P */
void insert(ListNode* n0, ListNode* P) {
    ListNode* n1 = n0->next;
    P->next = n1;
    n0->next = P;
}

  • 链表中删除结点：

    删除结点n0和n1之间的一个结点p，只需要使得n0->next指向n1，并delete p释放内存即可。

    /* 删除链表的结点 n0 之后的首个结点 */
void remove(ListNode* n0) {
    if (n0->next == nullptr)
        return;
    // n0 -> P -> n1
    ListNode* P = n0->next;
    ListNode* n1 = P->next;
    n0->next = n1;
    // 释放内存
    delete P;
}

链表的缺点在于，无法像数组一样直接访问任意结点，必须从头结点遍历到目标结点。如果要访问链表索引为i（即第i+1个）的结点，需要进行i次访问操作。并且链表占用的内存更多。

列表

List可以被理解成长度可变的数组，因此常常被称为动态数组。在C++中，vector即为此处描述的list。

/* 初始化 */
vector<int> list0; 	//无初始值
vector<int> list={ 1,3,2,5,4 }; 	//有初始值
/* 访问元素 */
int num = list[1]; //方式和数组相同
/* 更新元素 */
list[1] = 0;	//方式和数组相同
/* 清空列表 */
list.clear();	//直接调库很方便
/* 尾部添加元素 */
list.push_back(1);	//在尾部插入数字1
list.push_back(3); 	//在尾部插入数字3
/* 中间插入元素 */
list.insert(list.begin() + 3, 6);	//在索引为3处插入数字6
/* 删除元素 */
list.erase(list.begin() + 3);	//删除索引3处的元素
//遍历列表有两种方法
/* 通过索引遍历列表 */
int count = 0;
for (int i = 0; i < list.size(); i++) {
    count++;
}
/* 直接遍历列表元素 */
count = 0;
for (int n : list) {
    count++;
}
/* 拼接列表 */
list.insert(list.end(), list1.begin(), list.end());	//把list1拼接到了list之后
/* 排序列表 */
sort(list.begin(), list.end()); //从小到大排序

栈与队列

栈

Stack是先入后出（FILO）的线性数据结构。

/* 初始化栈 */
stack<int> stack;
/* 元素入栈 */
stack.push(1);
stack.push(3);
/* 访问栈顶元素 */
int top = stack.top();
/* 元素出栈 */
stack.pop();
/* 获取栈的长度 */
int size = stack.size();
/* 判断是否为空 */
bool empty = stack.empty();

栈的实现：

  • 用链表实现
    • 入栈用头插法
    • 出栈删除头结点即可
  • 用列表（动态数组）实现

队列

Queue是先入先出（FIFO）的线性数据结构。

/* 初始化队列 */
queue<int> queue;
/* 元素入队 */
queue.push(1);
queue.push(3);
/* 访问队首元素 */
int front = queue.front();
/* 元素出队 */
queue.pop();
/* 获取队列的长度 */
int size = queue.size();
/* 判断队列是否为空 */
bool empty = queue.empty();

队列的实现：

  • 用链表实现

    将头结点和尾结点分别视为队首和队尾，队首只能删除结点，队尾只能添加结点。

  • 用数组实现

    借助变量front来指向队首元素的索引，维护变量queSize来记录队列长度。
